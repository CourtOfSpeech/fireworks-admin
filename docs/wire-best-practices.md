## 概述

Google Wire 是一个编译时依赖注入工具，与运行时 DI 框架不同，Wire 在编译时生成依赖连接代码，确保类型安全且零运行时开销。

### Wire 的优势

- **编译时检查**：依赖关系在编译时验证，避免运行时错误
- **类型安全**：生成的代码完全类型安全
- **零运行时开销**：生成的代码与手写代码性能相同
- **代码可读**：生成的代码清晰易读，便于调试

## 核心概念

Wire 有两个核心概念：**Provider** 和 **Injector**。

### Provider（提供者）

Provider 是一个可以产生值的普通函数，通常是构造函数或工厂函数。

```go
// Provider 示例：创建数据库连接
func NewDatabase(cfg *Config) (*sql.DB, error) {
    return sql.Open("postgres", cfg.DSN())
}

// Provider 示例：创建仓库
func NewUserRepository(db *sql.DB) *UserRepository {
    return &UserRepository{db: db}
}

// Provider 示例：创建服务
func NewUserService(repo *UserRepository) *UserService {
    return &UserService{repo: repo}
}
```

### Injector（注入器）

Injector 是一个声明依赖关系的函数，Wire 会根据 Provider 自动生成实现代码。

```go
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewConfig,
        NewDatabase,
        NewUserRepository,
        NewUserService,
        NewApp,
    )
    return nil, nil
}
```

### wire.Build

`wire.Build` 是 Injector 函数的核心，它接收一组 Provider，Wire 会自动分析依赖关系并生成连接代码。

```go
wire.Build(
    // 单个 Provider 函数
    NewConfig,
    NewDatabase,
    
    // ProviderSet（一组 Provider）
    RepositorySet,
    ServiceSet,
    
    // 接口绑定
    wire.Bind(new(UserRepository), new(*userRepositoryImpl)),
    
    // 固定值
    wire.Value(Config{Env: "dev"}),
    
    // 结构体自动填充
    wire.Struct(new(App), "*"),
)
```

## 基础用法

### 最简示例

```go
package main

import (
    "fmt"
    "github.com/google/wire"
)

// 定义类型
type Message string
type Greeter struct {
    Message Message
}
type Event struct {
    Greeter *Greeter
}

// Provider 函数
func NewMessage() Message {
    return Message("Hello, Wire!")
}

func NewGreeter(m Message) *Greeter {
    return &Greeter{Message: m}
}

func NewEvent(g *Greeter) *Event {
    return &Event{Greeter: g}
}

// 业务方法
func (e *Event) Start() {
    fmt.Println(e.Greeter.Message)
}

// Injector 定义（注意 build tag）
//go:build wireinject

func InitializeEvent() *Event {
    wire.Build(
        NewMessage,
        NewGreeter,
        NewEvent,
    )
    return nil
}

// main.go
func main() {
    e := InitializeEvent()
    e.Start() // 输出: Hello, Wire!
}
```

### 生成的代码

运行 `wire` 后生成的 `wire_gen.go`：

```go
// Code generated by Wire. DO NOT EDIT.
//go:generate go run -mod=mod github.com/google/wire/cmd/wire
//go:build !wireinject

package main

func InitializeEvent() *Event {
    message := NewMessage()
    greeter := NewGreeter(message)
    event := NewEvent(greeter)
    return event
}
```

```go
        Addr: cfg.RedisAddr,
```

## 高级用法

### wire.Bind - 接口绑定

将接口绑定到具体实现，实现依赖反转：

```go
// 定义接口
type UserRepository interface {
    FindByID(ctx context.Context, id int64) (*User, error)
    Save(ctx context.Context, user *User) error
}

// 具体实现（注意：结构体名小写，不导出）
type userRepository struct {
    db *sql.DB
}

func NewUserRepository(db *sql.DB) *userRepository {
    return &userRepository{db: db}
}

func (r *userRepository) FindByID(ctx context.Context, id int64) (*User, error) {
    // 实现细节...
}

func (r *userRepository) Save(ctx context.Context, user *User) error {
    // 实现细节...
}

// 在 Injector 中绑定
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewDatabase,
        NewUserRepository,
        // 绑定接口到实现
        wire.Bind(new(UserRepository), new(*userRepository)),
        NewUserService,  // UserService 依赖 UserRepository 接口
        NewApp,
    )
    return nil, nil
}
```

**生成的代码：**

```go
func InitializeApp() (*App, error) {
    db, err := NewDatabase()
    if err != nil {
        return nil, err
    }
    userRepository := NewUserRepository(db)
    userService := NewUserService(userRepository) // 自动注入接口
    app := NewApp(userService)
    return app, nil
}
```

### 多接口绑定

一个实现可以绑定到多个接口：

```go
type Reader interface {
    Read() string
}

type Writer interface {
    Write(data string)
}

type ReadWriter interface {
    Reader
    Writer
}

type readWriterImpl struct {
    // ...
}

func NewReadWriter() *readWriterImpl {
    return &readWriterImpl{}
}

//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewReadWriter,
        wire.Bind(new(Reader), new(*readWriterImpl)),
        wire.Bind(new(Writer), new(*readWriterImpl)),
        wire.Bind(new(ReadWriter), new(*readWriterImpl)),
        NewApp,
    )
    return nil, nil
}
```

### wire.Value - 注入固定值

注入常量或预构建的值：

```go
type Config struct {
    Host string
    Port int
}

//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        // 注入值类型
        wire.Value(Config{Host: "localhost", Port: 8080}),
        NewServer,
        NewApp,
    )
    return nil, nil
}
```

注入指针值：

```go
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        // 注入指针值
        wire.Value(&Config{Host: "localhost", Port: 8080}),
        NewServer,
        NewApp,
    )
    return nil, nil
}
```

### wire.InterfaceValue - 注入接口值

注入满足接口的值：

```go
type Logger interface {
    Log(msg string)
}

type consoleLogger struct{}

func (l *consoleLogger) Log(msg string) {
    fmt.Println(msg)
}

//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        // 注入接口值
        wire.InterfaceValue(new(Logger), &consoleLogger{}),
        NewService,
        NewApp,
    )
    return nil, nil
}
```

### wire.Struct - 自动填充结构体

自动填充结构体字段：

```go
type App struct {
    DB     *sql.DB
    Cache  *redis.Client
    Logger Logger
}

//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewDatabase,
        NewRedisClient,
        NewLogger,
        // 指定要填充的字段
        wire.Struct(new(App), "DB", "Cache", "Logger"),
    )
    return nil, nil
}
```

使用 `"*"` 填充所有字段：

```go
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewDatabase,
        NewRedisClient,
        NewLogger,
        // 填充所有字段
        wire.Struct(new(App), "*"),
    )
    return nil, nil
}
```

### wire.FieldsOf - 从结构体提取字段

从配置结构体中提取特定字段：

```go
type Config struct {
    Database DatabaseConfig
    Redis    RedisConfig
    Server   ServerConfig
}

type DatabaseConfig struct {
    DSN string
}

type RedisConfig struct {
    Addr string
}

//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        LoadConfig,
        // 从 Config 中提取字段作为独立依赖
        wire.FieldsOf(new(*Config), "Database"),
        wire.FieldsOf(new(*Config), "Redis"),
        NewDatabase,    // 接收 DatabaseConfig
        NewRedisClient, // 接收 RedisConfig
        NewApp,
    )
    return nil, nil
}
```

## 清理函数

当 Provider 创建的资源需要清理时（如数据库连接、文件句柄），可以返回清理函数：

### 基本用法

```go
// Provider 返回清理函数
func NewDatabase(cfg *Config) (*sql.DB, func(), error) {
    db, err := sql.Open("postgres", cfg.DSN())
    if err != nil {
        return nil, nil, err
    }
    
    cleanup := func() {
        db.Close()
    }
    
    return db, cleanup, nil
}

// Injector 签名也要包含清理函数
//go:build wireinject

func InitializeApp() (*App, func(), error) {
    wire.Build(
        NewConfig,
        NewDatabase,
        NewApp,
    )
    return nil, nil, nil
}
```

### 生成的代码

```go
func InitializeApp() (*App, func(), error) {
    config := NewConfig()
    db, cleanup, err := NewDatabase(config)
    if err != nil {
        return nil, nil, err
    }
    app := NewApp(db)
    return app, func() {
        cleanup()
    }, nil
}
```

### 多个清理函数

Wire 会自动组合多个清理函数：

```go
func NewDatabase(cfg *Config) (*sql.DB, func(), error) {
    db, err := sql.Open("postgres", cfg.DSN())
    if err != nil {
        return nil, nil, err
    }
    return db, func() { db.Close() }, nil
}

func NewRedisClient(cfg *Config) (*redis.Client, func(), error) {
    client := redis.NewClient(&redis.Options{Addr: cfg.RedisAddr})
    return client, func() { client.Close() }, nil
}

func NewFileLogger(path string) (*FileLogger, func(), error) {
    f, err := os.OpenFile(path, os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
    if err != nil {
        return nil, nil, err
    }
    logger := &FileLogger{file: f}
    return logger, func() { f.Close() }, nil
}

//go:build wireinject

func InitializeApp() (*App, func(), error) {
    wire.Build(
        NewConfig,
        NewDatabase,
        NewRedisClient,
        NewFileLogger,
        NewApp,
    )
    return nil, nil, nil
}
```

### 使用清理函数

```go
func main() {
    app, cleanup, err := InitializeApp()
    if err != nil {
        log.Fatal(err)
    }
    defer cleanup() // 确保资源被清理
    
    if err := app.Run(); err != nil {
        log.Fatal(err)
    }
}
```

## 错误处理

Provider 可以返回错误，Wire 会自动处理错误传播：

### Provider 返回错误

```go
func NewDatabase(cfg *Config) (*sql.DB, error) {
    db, err := sql.Open("postgres", cfg.DSN())
    if err != nil {
        return nil, err
    }
    
    // 验证连接
    if err := db.Ping(); err != nil {
        return nil, fmt.Errorf("database ping failed: %w", err)
    }
    
    return db, nil
}

func NewConfig() (*Config, error) {
    cfg, err := LoadConfigFromFile("config.yaml")
    if err != nil {
        return nil, fmt.Errorf("failed to load config: %w", err)
    }
    
    if err := cfg.Validate(); err != nil {
        return nil, fmt.Errorf("config validation failed: %w", err)
    }
    
    return cfg, nil
}
```

### Injector 错误处理

```go
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        NewConfig,
        NewDatabase,
        NewRepository,
        NewService,
        NewApp,
    )
    return nil, nil
}

// 生成的代码会正确处理错误
func InitializeApp() (*App, error) {
    config, err := NewConfig()
    if err != nil {
        return nil, err
    }
    
    db, err := NewDatabase(config)
    if err != nil {
        return nil, err
    }
    
    repository := NewRepository(db)
    service := NewService(repository)
    app := NewApp(service)
    
    return app, nil
}
```

### 错误与清理函数组合

```go
//go:build wireinject

func InitializeApp() (*App, func(), error) {
    wire.Build(
        NewConfig,
        NewDatabase,
        NewRedisClient,
        NewApp,
    )
    return nil, nil, nil
}

// 生成的代码
func InitializeApp() (*App, func(), error) {
    config, err := NewConfig()
    if err != nil {
        return nil, nil, err
    }
    
    db, cleanup, err := NewDatabase(config)
    if err != nil {
        return nil, nil, err
    }
    
    redis, cleanup2, err := NewRedisClient(config)
    if err != nil {
        cleanup() // 清理已创建的资源
        return nil, nil, err
    }
    
    app := NewApp(db, redis)
    return app, func() {
        cleanup2()
        cleanup()
    }, nil
}
```

### 避免循环依赖

循环依赖通常意味着设计有问题。解决方案：

```go
// 错误：循环依赖
// ServiceA 依赖 ServiceB，ServiceB 又依赖 ServiceA

// 解决方案：引入事件或回调
type EventBus interface {
    Publish(event Event)
    Subscribe(handler EventHandler)
}

type ServiceA struct {
    eventBus EventBus
}

type ServiceB struct {
    eventBus EventBus
}

// 通过事件总线解耦
```

### 保持 Provider 纯粹

Provider 函数应该只负责创建对象，不应包含业务逻辑：

```go
// 错误：Provider 包含业务逻辑
func NewUserService(repo UserRepository) *UserService {
    service := &UserService{repo: repo}
    // 错误：在 Provider 中执行业务逻辑
    service.Initialize()
    service.LoadCache()
    return service
}

// 正确：Provider 只创建对象
func NewUserService(repo UserRepository) *UserService {
    return &UserService{repo: repo}
}

// 业务逻辑在应用启动时执行
func main() {
    app, cleanup, err := di.InitializeApp()
    if err != nil {
        log.Fatal(err)
    }
    defer cleanup()
    
    // 在这里执行初始化逻辑
    if err := app.UserService.Initialize(); err != nil {

    }
```

### Q: wire_gen.go 需要提交到 Git 吗？

**建议提交**。理由：

- 不需要 CI/CD 环境安装 wire 工具
- 便于代码审查

### Q: 如何处理可选依赖？

Wire 不直接支持可选依赖。推荐方案：

```go
// 方案1：使用空实现
type Logger interface {
    Log(msg string)
}

type noopLogger struct{}

func (l *noopLogger) Log(msg string) {}

func ProvideLogger() Logger {
    if os.Getenv("LOG_ENABLED") == "false" {
        return &noopLogger{}
    }
    return NewZapLogger()
}

// 方案2：使用函数选项
type ServiceOption func(*Service)

func WithLogger(logger Logger) ServiceOption {
    return func(s *Service) {
        s.logger = logger
    }
}

func NewService(repo Repository, opts ...ServiceOption) *Service {
    s := &Service{repo: repo}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

### Q: 如何处理相同类型的不同实例？

```go
// 使用类型别名区分
type PrimaryDB *sql.DB
type ReplicaDB *sql.DB

func NewPrimaryDB(cfg *Config) (PrimaryDB, error) {
    return sql.Open("postgres", cfg.PrimaryDSN)
}

func NewReplicaDB(cfg *Config) (ReplicaDB, error) {
    return sql.Open("postgres", cfg.ReplicaDSN)
}

func InitializeApp() (*App, error) {
    wire.Build(
        NewPrimaryDB,
        NewReplicaDB,
        NewApp,
    )
    return nil, nil
}
```

## 最佳实践总结

1. **提交生成的代码** - 将 `wire_gen.go` 提交到版本控制
2. **使用接口绑定** - 通过 `wire.Bind` 实现依赖反转
3. **组织 ProviderSet** - 按层级组织 Provider，便于管理
4. **返回清理函数** - 对于需要清理的资源，务必返回清理函数
5. **处理错误** - Provider 应该返回错误而不是 panic
6. **保持 Provider 纯粹** - Provider 只负责创建对象
7. **使用 wire.Struct** - 简化依赖容器的构建
8. **为测试创建专用 Injector** - 方便 Mock 和测试
9. **避免循环依赖** - 通过事件总线或回调解耦
10. **统一目录结构** - 将 Wire 相关代码集中在 `internal/di` 目录
